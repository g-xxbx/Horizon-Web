## [Telegram 的 t.me 域名因法律调查被暂停](https://www.whois.com/whois/t.me) ⭐️ 8.0/10

Telegram 的 t.me 短链接域名被暂停，ICANN 状态码如 clientRenewProhibited 表明涉及法律纠纷或准备删除。 全球数百万 t.me 链接正在使用，此次暂停中断了访问，并凸显了依赖 GoDaddy 等集中式域名注册商的风险。 域名显示 serverDeleteProhibited 和 clientRenewProhibited 状态，这些状态罕见且通常与法律纠纷相关。Telegram 使用 GoDaddy 作为注册商因其缺乏透明度而受到批评。

hackernews · Tiberium · 7月13日 19:52 · [社区讨论](https://news.ycombinator.com/item?id=48897878)

## [洛杉矶警察局终止与监控巨头 Flock 的合同](https://techcrunch.com/2026/07/13/lapd-lets-contract-with-surveillance-giant-flock-expire-citing-serious-concerns-over-civil-liberties-and-privacy/) ⭐️ 8.0/10

洛杉矶警察局（LAPD）于 2026 年 7 月终止了与 Flock Safety 的合同，理由是严重关切公民自由和隐私问题。然而，Flock 仍拥有摄像头和立杆的所有权，可以继续收集数据并出售给其他机构。 这一决定反映了公众对大规模监控的抵制日益增强，但也凸显了拆除设计上具有弹性的监控基础设施的挑战。此案强调了加强数据所有权监管以及限制政府获取私人收集数据的必要性。 Flock Safety 拥有摄像头和立杆的所有权，因此即使合同终止，数据收集仍在继续。LAPD 仍可通过致电 Flock 或通过其他机构（如加州公路巡警、洛杉矶县警局或 FBI）获取数据，这削弱了合同到期本应带来的隐私保护。

hackernews · forks · 7月13日 15:11 · [社区讨论](https://news.ycombinator.com/item?id=48893947)

## [DOOMQL：基于 SQLite SQL 查询的类 Doom 游戏](https://simonwillison.net/2026/Jul/13/doomql/#atom-everything) ⭐️ 8.0/10

Peter Gostev 发布了 DOOMQL，这是一款类似 Doom 的游戏，使用 SQLite 作为整个游戏引擎，通过 SQL 查询控制移动、敌人和渲染。该游戏作为 Python 终端脚本运行，并包含使用递归 CTE 在 SQL 中实现的射线追踪功能。 DOOMQL 极具创意，突破了 SQLite 的能力边界，将数据库引擎重新用于实时游戏渲染和逻辑处理。它挑战了人们对数据库能力的传统认知，展示了 SQL 与交互式应用的新型整合方式。 游戏将全部状态存储在 /tmp/doomql/.doomql/doomql.sqlite 的 SQLite 数据库中，可使用 Datasette 进行探索。渲染部分包含一个通过递归 CTE 在单个大型 SQL 查询中实现的完整射线追踪器。

**背景**: SQLite 是一款自包含、无需服务器、零配置的 SQL 数据库引擎，广泛嵌入于各类应用中。该项目利用 SQLite 的递归公用表表达式（CTE）和虚拟表来处理游戏逻辑与实时像素渲染，展示了数据库可用于远超数据存储范围的任务。

## [CoT 是扩展陷阱，潜在推理与 BDH 崛起](https://www.reddit.com/r/MachineLearning/comments/1uviru5/chain_of_thought_is_a_scaling_trap_the_next_wave/) ⭐️ 8.0/10

一篇 Reddit 帖子认为链式思维推理是一种扩展陷阱，存在忠实性和成本问题，并提倡潜在推理方法，如 Coconut、RecursiveMAS 和 BDH。该帖子还指出了潜在推理的可解释性差距，并提出了一个符号化的外层治理层用于审计。 这一批评挑战了在语言模型中使用链式思维推理的主流范式，提出转向潜在推理，这可以降低成本并提高忠实性。讨论还强调了需要新的治理框架来维护模型内部推理时的问责制。 Coconut 在潜在空间中实现广度优先搜索，BDH 在无需 CoT 的情况下在 Sudoku Extreme 上达到 97.4% 的 top-1 准确率，RecursiveMAS 使用潜在嵌入进行智能体通信。该帖子指出时间递归是流式环境中潜在推理的挑战，并建议使用基于 DAG 的规划器和确定性验证的外层循环。

reddit · r/MachineLearning · /u/meowsterpieces · 7月13日 17:50

**背景**: 链式思维（CoT）推理提示模型生成逐步解释，这可以提高性能，但也增加了 token 使用量，并可能产生不忠实的原因。潜在推理方法旨在在不生成中间文本的情况下进行内部计算，从而可能降低成本和提升效率。然而，这种不透明性在高风险应用中引发了担忧，因此提出了结合符号规划与神经计算的混合治理层方案。
