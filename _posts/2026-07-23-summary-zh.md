## [PyPI 禁止向 14 天以上旧版本上传新文件](https://simonwillison.net/2026/Jul/23/seth-larson/#atom-everything) ⭐️ 8.0/10

PyPI 自 2026 年 7 月 22 日起拒绝向任何超过 14 天的旧版本上传新文件。该变更通过 Pull Request #19727 实施，旨在防止因令牌泄露导致的供应链攻击。 这一措施封堵了潜在的后门，攻击者可能利用泄露的发布令牌向用户信任的长期旧版本注入恶意代码。这是对整个 Python 包生态系统的主动安全改进。 该限制适用于所有超过 14 天的版本，无论是否已有文件。维护者仍可在创建后 14 天内更新版本，对于旧版本则只需创建新的发行版。

rss · Simon Willison · 7月23日 04:50

**背景**: PyPI（Python 包索引）是 Python 的官方第三方软件仓库。软件供应链攻击指攻击者通过窃取 API 令牌等方式破坏构建或分发管道以注入恶意代码。传统上 PyPI 使用长期有效的 API 令牌，一旦泄露可能被利用。新规则缩短了此类攻击的机会窗口。

一位研究者在从 OpenReview 下载的 NeurIPS 2026 论文 PDF 中发现了一个隐藏的提示注入，这很可能是会议方为检测 LLM 生成的同行评审而插入的。 这一事件引发了人们对顶级 AI 会议同行评审诚信的严重担忧，因为它暗示了对审稿人的秘密监控以及对提示注入技术的潜在滥用。同时也凸显了在学术评审过程中检测 AI 生成内容日益增长的挑战。 隐藏的提示要求审稿人包含三个特定短语：'This work addresses the central challenge'、'The claims of the paper'和'Overall, I find this submission.' 研究者建议，如果审稿中出现所有这些短语，应向区域主席报告。
